Image Search Abstraction Layer

dmiten | GitHub

User Stories:
  • I can get the image URLs, alt text and page urls for a set of images relating to a given search string.
  • I can paginate through the responses by adding a page=2&limit=10 parameters to the URL.
  • I can get a list of the most recently submitted search strings.
    • Using API:
  • ../api/search is wrapper for Flickr API. It accept a string as parameter: ..api/search/{string}. Spaces must be replaced with %20. Flickr API will use this string for a free text search. Photos who's title, description or tags contain the text will be returned. You can exclude results that match a term by prepending it with a - character. Native Flickr pagination used. API accept start page number and limit per page as page=40&limit=500. Default values are page=1&limit=10.
    For example: ../api/search/cat%20sleep?page=40&limit=500 will return photos with "cat" and "sleep" in the tags or descriptions from page number 40 and 500 objects per page.
  • ../api/history doesn't need any parameters. It return 10 of the most recently submitted search strings.
